🎯 **What:**
Fixed a timing leak vulnerability in the `timingSafeEqual` function (`liftlogic/utils/security.ts`) that exited early if the compared strings had different lengths.

⚠️ **Risk:**
If left unfixed, attackers could infer the exact length of sensitive strings (like target hashes, passwords, or authentication tokens) by measuring response times. A shorter response time meant the length check failed early, allowing them to iterate until they determined the secret's correct length as a precursor to fully extracting it via subsequent timing attacks.

🛡️ **Solution:**
Removed the early return. The updated function records whether the lengths match in a variable. To ensure execution time is solely dependent on the length of the attacker-controlled string and avoid array out-of-bounds exceptions, it compares the first string against itself if lengths do not match, while retaining the false result. Thus, the loop always executes `a.length` times, nullifying the timing discrepancy.
