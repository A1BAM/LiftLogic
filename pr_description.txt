🎯 **What**
Extracted identical array chunking and database bulk insert logic found in both the `POST /bulk` and `POST /` (upsert) handlers in `worker.ts` into a unified `executeBulkInsert` asynchronous helper function.

💡 **Why**
To adhere to the DRY (Don't Repeat Yourself) principle and reduce the cyclomatic complexity within the main request handlers. Consolidating the logic eliminates the risk of future schema drift (where one endpoint gets updated while the other is missed) and drastically improves the readability of the routing blocks.

✅ **Verification**
Verified the refactor via automated test suites: ran `pnpm exec tsc --noEmit` and `pnpm test` which validated correct types and zero regressions in `worker.test.ts`. Verified the generated query logic and flow inside the new helper remain exactly as before.

✨ **Result**
Code duplication is eliminated and the total file footprint is reduced. The Cloudflare worker endpoint blocks are now focused purely on validation and routing, delegating the heavy database operations to a specialized, testable helper function.
